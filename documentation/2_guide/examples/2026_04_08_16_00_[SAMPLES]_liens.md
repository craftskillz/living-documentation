# Liens

Living Documentation supporte plusieurs types de liens : liens web classiques, navigation interne vers d'autres documents, ancres dans la page courante, et liens vers des fichiers joints. Tous s'écrivent en Markdown standard ou avec des shortcodes dédiés.

---

## Lien web externe

La syntaxe standard Markdown. S'ouvre dans un nouvel onglet.

**Syntaxe :**

```markdown
Voici la [documentation officielle de Svelte](https://svelte.dev/docs)
et le [dépôt GitHub de Living Documentation](https://github.com).
```

**Rendu :**

Voici la [documentation officielle de Svelte](https://svelte.dev/docs) et le [dépôt GitHub de Living Documentation](https://github.com).

---

## Lien vers un document interne

Utiliser le shortcode `[doc-link: chemin]` pour naviguer vers un autre document du projet sans recharger la page. Le chemin est relatif à la racine du dossier docs, sans extension `.md`.

**Syntaxe :**

```markdown
[doc-link: 2*guide/examples/2026_04_22_16_00*[SAMPLES]\_emojis]
[doc-link: 1_concepts/architecture | Voir l'architecture]
```

**Rendu :**

[doc-link: 2*guide/examples/2026_04_22_16_00*[SAMPLES]\_emojis]

[doc-link: 2*guide/examples/2026_04_22_16_00*[SAMPLES]\_emojis | Voir les emojis →]

---

## Lien ancre , même document

Pour sauter à une section dans la page courante. L'ancre est générée automatiquement à partir du texte du titre (minuscules, espaces → tirets, caractères spéciaux supprimés).

**Syntaxe :**

```markdown
[Aller au tableau de référence](#récapitulatif)
[Revenir en haut](#liens)
```

**Rendu :**

[Revenir en haut de cette page](#liens)

---

## Lien vers un fichier joint

Un fichier attaché au document se lie comme un lien classique, avec le chemin relatif.

**Syntaxe :**

```markdown
[Télécharger le rapport PDF](./rapport-q4-2025.pdf)
[Voir les données brutes](./assets/export.csv)
```

**Rendu :**

[Télécharger le rapport PDF](./rapport-q4-2025.pdf) _(lien exemple, fichier non inclus)_

---

## Lien email

**Syntaxe :**

```markdown
Contacter l'équipe : [support@example.com](mailto:support@example.com)
```

**Rendu :**

Contacter l'équipe : [support@example.com](mailto:support@example.com)

---

## Image cliquable (lien vers plein écran)

En enveloppant une image dans un lien pointant vers elle-même, un clic l'ouvre en mode lightbox plein écran.

**Syntaxe :**

```markdown
[![Schéma d'architecture](./schema.png)](./schema.png)
```

**Rendu :**

L'image s'affiche normalement, et un clic l'ouvre en plein écran dans Living Documentation.

---

## Récapitulatif

| Type             | Syntaxe                  | Comportement                       |
| ---------------- | ------------------------ | ---------------------------------- |
| Lien externe     | `[texte](https://...)`   | Nouvel onglet                      |
| Document interne | `[doc-link: chemin]`     | Navigation SPA (sans rechargement) |
| Ancre interne    | `[texte](#section)`      | Défilement dans la page            |
| Fichier joint    | `[texte](./fichier.ext)` | Téléchargement ou ouverture        |
| Email            | `[texte](mailto:...)`    | Ouvre le client mail               |
| Image lightbox   | `[![alt](img)](img)`     | Plein écran au clic                |
