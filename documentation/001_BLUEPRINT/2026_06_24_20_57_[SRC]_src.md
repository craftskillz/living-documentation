Le dossier `src/` contient le cœur applicatif de Living Documentation. C'est ici que vivent le serveur local, les API, le serveur MCP, la logique partagée et l'interface utilisateur.

Il sert à transformer un dossier de documentation Markdown en application locale consultable : lecture, édition, configuration, métadonnées, diagrammes, fichiers, contexte IA, Blueprint et autres écrans du produit.

L'objectif est simple : regrouper le code qui fait fonctionner l'application, depuis les routes HTTP jusqu'à l'expérience navigateur.

Vue d'ensemble : `server.ts` démarre l'application, `routes/` expose les fonctionnalités HTTP, `lib/` porte les règles partagées, `mcp/` ouvre les outils pour les agents IA, et `frontend-svelte/` contient l'interface utilisateur.

Fonctionnellement, ce dossier couvre cinq usages importants :

- servir l'application Living Documentation en local ;
- lire, écrire et organiser les documents Markdown ;
- exposer les fonctionnalités aux agents IA via MCP ;
- gérer les données locales du projet, comme la configuration, les métadonnées et les diagrammes ;
- afficher les écrans utilisateur avec le frontend Svelte.
