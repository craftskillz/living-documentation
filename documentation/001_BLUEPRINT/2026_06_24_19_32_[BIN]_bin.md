Le dossier `bin/` contient le point d'entrée exécutable du package npm `living-ai-documentation`. C'est la première couche appelée par `npx living-ai-documentation` et par `npm run start`.

Il sert de sas d'entrée, en vérifiant si un projet Living Documentation existe à l'endroit indiqué à l'aide du fichier `.living-doc.json`, puis en proposant un wizard en ligne de commande pour en initialiser un quand aucun dossier n'est fourni.

L'objectif est simple : permettre à l'utilisateur soit d'ouvrir une documentation existante, soit de créer rapidement un projet prêt à être consulté.

Vue d'ensemble : `cli.ts` porte cette expérience de démarrage, depuis le choix du dossier jusqu'au lancement de l'application locale.

Fonctionnellement, ce fichier couvre trois usages importants :

- démarrer une documentation existante ;
- initialiser un nouveau projet Living Documentation en anglais ou en français ;
- éviter d'écraser les fichiers d'orientation IA déjà présents dans le projet.
