# Generated OKF `log.md` changelog

Ticket 10. Completes the reserved-files pair started in
[index.md generation (T09)](?doc=ADRS%252F2026_07_10_07_48_%255BOKF%255D_generated_okf_indexmd_progressivedisclosure_listings_and_reserved_files).

## Decision
`src/lib/okf/log-generator.ts` `generateOkfLogFile(docsPath)` writes the reserved
`log.md` at the bundle root. It runs `git log --diff-filter=A --name-only` scoped
to the docs folder, groups the **added** `.md` concepts by commit date
(`## YYYY-MM-DD`, newest first) and lists `* **Added**: <title>`. Reserved and
non-`.md` files are skipped; a non-repo folder yields a minimal
"No history available" log. It is invoked by `migrateDocsFolder` right after the
index generation, so migration / `migrate` / init keep it fresh.

## Rationale
The autocommit messages are uniform ("docs: update living documentation"), so a
raw message dump is useless. Describing each date by the **concepts added** (title
from the filename, robust to later renames/deletions) is git-grounded and
genuinely informative.

## Consequences
- `documentation/log.md` is a date-grouped changelog of concept additions.
- Titles come from filenames (deterministic); reading the frontmatter `title` for
  still-present files is a possible future refinement.
- Like `index.md`, `log.md` is a reserved file excluded from the document list and
  from migration.
