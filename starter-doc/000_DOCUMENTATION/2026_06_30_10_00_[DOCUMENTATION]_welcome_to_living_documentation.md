### Welcome to **Living Documentation**!

Your local Markdown documentation workshop.

<!-- image-width: 2/3 -->

![Local **Living Documentation** workshop connecting Markdown documents, notes, processes, diagrams, Git, and AI agents](/images/DOCUMENTATION/concept-01-hero-produit.jpg)

This is not a code generation tool, but a tool for generating, maintaining, versioning, and automating documentation.

**Living Documentation** helps you keep useful knowledge in one place, in a readable and versionable format:

- Markdown documents organized by folders and categories
- meeting notes and reports
- procedures, processes, and checklists
- code documentation and architecture decisions (ADRs)
- diagrams linked to documents
- images, attachments, and generated visuals
- agent runs and automation traces
- Git history and version comparison

### A local-first model

<!-- image-width: 2/3 -->

![Local-first architecture connecting a Markdown folder, Git, a web interface, diagrams, and AI agents](/images/DOCUMENTATION/concept-03-local-first.jpg)

Your documentation is based on files stored in your local workspace:

- documents are Markdown files that can be read by you or by your LLMs
- versioning uses Git to record changes continuously and make all documents recoverable
- the tool can be used on its own, then extended with its MCP tools and internal agents

### Let's start the tour!

Before anything else, I recommend configuring Git integration in **Living Documentation** so you can benefit from automatic versioning of all your changes.
You need Git installed on your computer. Then, from the terminal, open the folder that contains **Living Documentation** and initialize a Git repository.

<!-- code-width: 1/4 -->

```bash
git init
```

Then go to the admin page (click the <kbd>Admin</kbd> menu with <kbd>Ctrl + click</kbd> or <kbd>Shift + click</kbd> to open a new window), configure Git integration, save, and come back to continue reading.

![image](/images/DOCUMENTATION/admin_git_integration.png)
